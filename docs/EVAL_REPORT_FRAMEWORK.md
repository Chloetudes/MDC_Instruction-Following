## 1. 执行摘要

### 1.1 背景与动机

- **行业痛点**：传统指令遵循基准（IFEval、CFBench）存在约束机械化、评分静态化、数据污染等问题，模型分数趋于饱和（85–95 分），难以区分真实复杂场景能力。
- **本报告定位**：提出并实施一套**多维约束评测框架**，构建 L1–L3 三级分类体系，采用 AdvancedIF Rubrics 五维度（D1–D5）二元 PASS/FAIL 评分，覆盖全自动化流水线。

### 1.2 核心结论（Bullet 化，便于快速扫读）

- **指令合成**：基于任务框架定义，数据泛化性强；引入基于模型能力的合适攻击策略，提升指令难度，降低构造成本。
- **指令评估**：任务框架将约束难度落实为可计算公式，可客观衡量任意指令复杂度，预判分布与难度。
- **更强区分度**：自建复杂指令集上，模型得分方差显著大于公开基准，有效打破「分数天花板」。业务仿真需求设计，体现真实场景复杂约束遵循程度，客观区分模型层级。
- **能力短板**：证实模型普遍「重格式、轻逻辑」。在格式约束（D4）上表现优异，在涉及多步推理的「流程步骤」对应维度（D2 操作执行逻辑）上得分普遍较低。
- **评分机制**：基于统一约束难度框架，设计 5 步分层约束评估标准（D1–D5），将语言理解、信息归纳、任务执行、形式化输出等维度分层聚合评估，提升评估准确度和效率。

## 2. 评测设计与方法论

### 2.1 核心理念与问题定义

针对三大痛点：**约束机械化**、**权重平均主义**、**缺乏结构化解析**，本框架确立：

1. **约束为核心**：将指令视为可验证约束的集合，通过解析约束结构重构逻辑。
2. **类型区分**：核心任务（流程步骤）与具体执行的约束要求权重远高于修饰性约束（格式、标点）；在 Rubric 设计时将约束类型映射到 D2–D5 维度，实现按类型聚焦。
3. **结构化解析**：L1–L3 三级分类将指令从平面文本解构为立体逻辑结构。

### 2.2 多维约束分类体系（L1–L3）

#### L1：指令结构层（逻辑拓扑）

- 约束间拓扑关系：单点、并列、链式、条件、嵌套。
- 决定任务规划难度；覆盖 QA、NLU、NLI、NLP、Role-playing 等。

#### L2：约束功能层（角色定位）

| L2 类型 | 说明 | 对应 D 维度（失分分析） |
|---------|------|--------------------------|
| 教学约束 | 前置知识与规则定义 | 一般不直接评分；若涉及理解分歧 → D1 |
| 素材约束 | 需处理的输入材料 | 一般不直接评分；若涉及理解 → D1 |
| **流程步骤** | 提取、分类、判断等核心操作 | **D2 操作执行逻辑** |
| 格式输出 | JSON、表格等结构化要求 | **D4 格式与形式化要求** |
| 边界范围 | 主题、风格、禁止项、来源限定 | **D3 边界范围限制** |
| 数量篇幅 | 字数、个数、条数限制 | **D3 边界范围**（数量）或 **D4**（篇幅/格式） |

#### L3：约束类型层（具体执行）

- 细粒度约束，支撑数据合成多样性；分析与评估侧重 L2。

### 2.3 约束类型失分与 D 维度对应分析

本报告**不做动态加权与连坐**，评分采用 **passed/total** 的均权聚合。失分分析时，将 **L2 约束类型** 与 **D 维度** 对应，便于从理论（约束设计）和模型能力（维度表现）两端对齐：

| D 维度 | 职责 | 口诀 | 主要对应的 L2 约束类型 | 分析视角 |
|--------|------|------|------------------------|----------|
| D1 业务理解 | 仅在有理解分歧时启用 | — | 教学、素材（理解类） | 题目背景/规则理解 |
| D2 操作执行逻辑 | 是否做了（完整性、顺序性） | 缺了→D2 | **流程步骤** | 逻辑执行、多步推理 |
| D3 边界范围限制 | 是否越界（禁止项、来源、范围、数量） | 超了→D3 | **边界范围、数量篇幅** | 越界、超限 |
| D4 格式与形式化要求 | 形式规范（结构、标识符、排版） | 歪了→D4 | **格式输出、数量篇幅（篇幅）** | 格式、结构 |
| D5 内容质量 | 做得对不对（准确性、深度） | 错了→D5 | 流程步骤（质量侧）、内容要求 | 准确性、深度 |

**分析示例**：若某题 D2 通过率明显低于 D4，可解读为模型在「流程步骤」类约束（逻辑执行）上弱于「格式输出」类约束，与「重格式、轻逻辑」结论一致。

### 2.4 全自动化评测流水线

| 阶段 | 产出 | 要点 |
|------|------|------|
| 阶段一：指令合成 | 复杂指令 | L3 任务意图表；HM 类型（人机协作）难度较高 |
| 阶段二：指令质量与难度 | difficulty_score、difficulty_level | 约束原子拆解、数据清洗 |
| 阶段三：评分标准生成 | Rubrics（D2–D5 检查点） | 五步分析框架、MCOAB 原则、专家级考点 |
| 阶段四：参考答案生成 | 满分回复 | 部分人工修正、强模型生成 |
| 阶段五：回复评估 | 结构化评估（FINAL_SCORE、rubrics_check） | 按指令分组、Prompt Caching |

### 2.5 评估方法论（与业界对齐）

- **评分公式**：`score = round(100 * passed / total)`，格式为 `"89% (24/27)"`。
- **MCOAB 原则**：MECE + Constructive + Objective + Atomic + Binary。
- **五维度框架（D1–D5）**：D1 业务理解 + D2 执行逻辑 + D3 边界限制 + D4 格式要求 + D5 内容质量。
- **原子化可验证**：参考 IFEval verifiable constraints；每条 rubric 仅 PASS/FAIL。
- **致命项（F）**：任一致命项 FAIL → SATISFIED_ALL_REQUIREMENTS = "NO"。
- **人机协作 Rubric**：参考 AdvancedIF，专家洞察价值考点，维度框架标准化。
- **模型能力基线**：2024–2025 基线中，基本都能做到常见格式、简单抽取；易错点集中于嵌套指令、否定遵守、条件判断等。

## 4. 维度与能力细分

### 4.1 D2–D5 维度通过率（雷达图/热力图）

- **流程步骤 vs 格式输出**：对应 D2 vs D4。几乎所有模型在 D4 通过率显著高于 D2，验证「重格式、轻逻辑」。
- **边界范围（D3）**：与对齐训练强度相关；闭源 SOTA 在非严格约束表现更好。
- **难度定级效度**：随 difficulty_level 从 E→S 升高，模型性能下滑；高难度区间（A/S）头部模型仍保持较高通过率，证明难度区分有效。

_数据来源：`analysis/rubric_dimension_analysis.py` dimension_pivot_df_

### 4.2 约束类型失分与 D 维度对应分析（理论—实证对齐）

将 L2 约束类型与 D 维度对应后，可从失分分布验证模型能力短板：

| 现象 | 理论解释 | 实证佐证 |
|------|----------|----------|
| D2 通过率 < D4 | 流程步骤类约束（逻辑执行）难于格式类约束 | 各模型 D2/D4 通过率对比表 |

## 5. 典型案例深度分析

### 5.1 公开数据案例：「高分低能」成因

- **案例来源**：source=to_b / nlp_* 等公开数据。
- **传统评测**：格式正确即可满分。
- **本框架分析**：D2 流程步骤细粒度评分暴露语义理解浅层化；D3/D4 高分、D2 低分体现「形式大于内容」。

### 5.2 自建数据案例：逻辑漏洞精准捕捉

- **案例来源**：source=H/R/HM/M 等自建数据。
- **模型 A（SOTA）**：D2 高分，精准提取态度转折。
- **模型 B（开源）**：D2 低分，罗列未提炼；D4 格式正确无法掩盖 D2 逻辑缺陷。
- **约束类型↔维度**：本题 L2 以流程步骤为主，失分集中在 D2，与理论一致。

### 5.3 案例呈现规范（与 REPORT_PREVIEW_SAMPLE 对齐）

每题宜含：

- **题目内容**（摘要或全文）
- **L1 / L2 / L3**、**难度等级**、**source**
- **各模型得分**（与本题最佳对比）
- **维度得失分**（D2/D3/D4/D5 通过率）
- **专家意见**（若有）
- **AI 综合评估**（LLM 归纳）
- **失分点分析**（按 Dx_Y 条目，对应 L2 约束类型）

## 6. 结论与建议

### 6.1 总结

1. **理论创新**：L1–L3 三级分类，指令立体化解构；约束类型与 D 维度对应，支撑理论—实证对齐分析。
2. **方法创新**：AdvancedIF Rubrics 五维度、二元 PASS/FAIL、致命项机制，打击「形式大于内容」虚假高分。
3. **工程实现**：全自动化流水线 + Prompt Caching，高效低成本。

### 6.2 对模型训练的建议

- 增加高权重数据：复杂推理链、多步规划（流程步骤类）比例。
- 强化 D2 相关数据：减少单纯格式模板即可完成的低质量指令。
- 去伪存真：聚焦有区分度、易出错、有挑战性的价值考点。

---

## 7. 局限性与未来工作

### 7.1 局限性

- **成本**：全流程 LLM-as-a-Judge，相比正则匹配成本更高。
- **主观边界**：「风格」「语气」等约束判定存在主观性，需优化评估 Prompt 一致性。
- **人机一致性**：LLM 裁判仍低于人人一致性，建议结合专家抽检（见 EVAL_INDUSTRY_COMPARISON 2.1）。

### 7.2 未来工作规划

1. **动态评测演进**：类似 DyVal，根据模型反馈动态调整题目难度；时效性素材、身份防伪。
2. **多模态扩展**：约束体系扩展至图像、视频。
3. **长上下文与 Agent**：超长文本分析、多步工具调用专用评测。

### 厂商专项报告说明

厂商专项报告（`generate_series_reports` 阶段）结构与主体报告对齐，侧重厂商视角：
- **1. 系列总览**：本系列模型排名、与各系列横向对比（Mann-Whitney U 显著性）
- **2. 按数据来源分组**：本系列在公开/自建数据上的得分（与主体 source 分组一致）
- **3. 版本纵向对比**：系列内版本排名、题目×版本得分矩阵、L1 维度对比
- **4. 系列横向对比**：整体排名、L1/难度维度各系列对比
- **5. 维度深度分析**：L1/L2/L3/难度各维度的本系列均分 vs 全体均分
- **6. D 维度得失分**：本系列各模型 D1–D5 通过率（约束类型↔D 维度对应）
- **7. 优劣势分析**：L2 维度相对优势/劣势 TOP10
- **8. 厂商视角小结**：本系列均分、与全体差距、系列内最佳、改进建议
- **9. 典型案例**：本系列表现优于全体均分的题目（若有 2–5 个则展示；无则不写）
- **10. Badcase 案例分析**（重点）：指明本系列在哪些题目、哪些维度上容易失分，含题目内容、本系列各模型得分、维度通过情况、失分点分析（基于 rubrics_check 解析 D2/D3/D4/D5 及具体子项 FAIL 原因）

输出格式：`series_report_{系列名}.xlsx`、`series_report_{系列名}.md`。

## 使用建议

1. **撰写顺序**：先完成 2（方法论）和 8（附录），再写 3–5（结果与案例），最后写 1（摘要）和 6–7（结论与局限）。
2. **数据填充**：自动报告已从 `analyze_results`、`dimension_stats`、ranking、source_stats 等填充；手工报告时参考「自动报告与数据映射」表。
3. **约束类型↔维度分析**：失分分析时优先按 D2/D3/D4/D5 归因，并对应回 L2 约束类型，实现理论与模型能力双重对齐。
4. **图表规范**：热力图、雷达图需标注数据来源与批次；表格需列明样本量。
