# 自动化评测与数据合成系统 — 流程与架构概览

> 本文档为对外发布用：概述系统流程与架构，便于快速理解与复现。详细设计见 [ARCHITECTURE.md](ARCHITECTURE.md)。

---

## 系统定位

本系统是一套**复杂指令遵循能力自动化评测与数据合成**框架，实现从题目与评分标准、到多模型回复、裁判评分、维度化分析、直至可视化报告与**数据合成建议**的全流程自动化，支持按数据来源（公开/自建）切换统计口径，用于模型迭代与数据建设。

---

## 核心流程（端到端）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  题目与约束                                                                   │
│  (questions + L1/L2/L3 + source + 难度)                                      │
└───────────────────────────────────┬─────────────────────────────────────────┘
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 0 / 0.5 / 1 / 1.5（可选）                                             │
│  指令生成 → 指令提取 → 质量评估 → 评分标准生成                                 │
└───────────────────────────────────┬─────────────────────────────────────────┘
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 2 & 3                                                                 │
│  参考答案生成 → 多模型回复生成 → replies.xlsx                                 │
└───────────────────────────────────┬─────────────────────────────────────────┘
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 4                                                                     │
│  裁判评估 → eval_{batch_id} + eval_{batch_id}_raw（含 D1–D5 检查点）          │
└───────────────────────────────────┬─────────────────────────────────────────┘
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  analyze_results                                                             │
│  排名 / 维度失分率 / L1 失分画像 / 人机一致性 / 题目质量分析                    │
└───────────────────────────────────┬─────────────────────────────────────────┘
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  generate_report                                                             │
│  Markdown & HTML 报告 + 数据合成建议（基于失分与维度分布）                      │
└─────────────────────────────────────────────────────────────────────────────┘
```

---

## 架构要点

| 维度 | 说明 |
|------|------|
| **阶段化 Pipeline** | 各 Stage 独立可跑、可配置顺序，支持断点续跑与缓存；配置以 `outputs/<project_id>/config.json` 为主。 |
| **五维度评分（D1–D5）** | 业务理解、流程步骤、边界范围、格式形式、内容质量；从裁判 raw 解析检查点，汇总维度失分率与 L1 失分特征。 |
| **统计口径可配置** | 全量 / 仅公开 / 仅自建 / 自定义 source，便于对比公开基准与自建数据表现。 |
| **数据合成建议** | 基于模型失分画像与维度分布，输出可执行的数据建设建议，形成「评测 → 分析 → 数据迭代」闭环。 |
| **无敏感信息发布** | 代码与文档中不包含 API Key、内部 URL；配置通过 `config.example.py` 与 `config.example.json` 提供模板。 |

---

## 代码与配置入口

- **运行入口**：`python -m evaluation.main`（读取项目 config 与 stages）。
- **配置模板**：`config.example.py`（复制为 `config.py` 并填写 key）、`config.example.json`（复制到 `outputs/<project_id>/config.json`）。
- **数据格式**：题目表/回复表字段说明见 [DATA_SCHEMA.md](DATA_SCHEMA.md)；阶段与 preset 见 [STAGES_REFERENCE.md](STAGES_REFERENCE.md)。
