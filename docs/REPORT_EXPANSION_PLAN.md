## 二、数据与能力映射

### 2.1 当前数据源可支撑的扩充

| 数据源 | 字段/结构 | 可支撑的扩充 |
|--------|----------|-------------|
| `replies` | eval_score, model, qid | 分数分布、按意图/难度分组 |
| `questions` | L1, L2, L3, difficulty_level, source, query | 意图级分析、难度分组 |
| `eval_*_raw` | rubrics_check (D2_1, D3_1, ... PASS/FAIL) | 约束级失分、维度通过率 |
| `row_level_df` | qid, model, D2_pass, D2_total, ... | 题目×模型 维度通过率 |
| `evaluation_criteria` / 约束库 | 约束 ID、内容、类型、权重 | 约束效能分级、约束类型汇总 |
| `reference` | 参考答案 | 与参考答案对比（若存在） |

### 2.2 参考报告中的「约束失分详情」与我们的对应关系

参考报告格式示例：

```
| 约束 | 类型 | 满分 | 模型A | 模型B | 模型C |
|------|------|------|-------|-------|-------|
| 拆解部件结构... | 流程步骤 | 4.0 | 0.0 ❌ | 11.1 ✓ | 11.1 ✓ |
```

**对应关系**：

- 参考报告中的「约束」≈ 我们的 **rubrics_check 检查点**（D2_1, D3_1, ...）或 **evaluation_criteria 中的约束条目**
- 若 evaluation_criteria 可解析为「约束 ID → 类型 → 满分」，则可构建约束×模型得分表
- 若仅有 rubrics_check：可将 D2_1、D3_2 等视为「原子约束」，类型由维度标签（D2→流程步骤、D4→格式形式）推导

**实现策略**：

1. **优先方案**：从 `rubrics_check` 解析出各检查点 PASS/FAIL，按 D 维度聚合，形成「维度×模型」通过率表；若 evaluation_criteria 可结构化解析，再扩展到「约束 ID × 模型」细粒度表
2. **降级方案**：仅输出「D2/D3/D4/D5 × 模型」的通过率，辅以 AI 分析中的失分点描述

## 三、分阶段扩充方案

### 阶段一：低成本、高价值（优先实现）

#### 1.1 分数分布统计（0–20, 20–40, 40–60, 60–80, 80–100）

**目标**：在整体概况中增加「各模型分数分布」，与参考报告 1.2 一致。

**实现**：

- 新增 `analysis/score_distribution.py` 或于 `ranking.py` 中增加 `compute_score_distribution(replies_df)` 
- 对每个 model，统计各分数段的题目数及占比
- 在 `report_writer_md.py`、`report_writer_html.py` 的「整体概况」后插入分数分布表/图

**数据流**：`replies` → groupby model → 按 eval_score 分段 → 汇总

#### 1.2 意图级别分析（L2 或 L3 最小颗粒度）

**目标**：输出「各意图：题目数、平均分数方差、最佳/最差模型、各模型均分」，与参考报告二一致。

**实现**：

- `ranking.py` 已有 `analyze_l1_dimension`、`analyze_l2_dimension` 等
- 新增或扩展示例：`analyze_intent_level(replies_with_q, dim_col='L2')` 返回汇总表
- 在报告中增加「意图级别分析」章节，遍历 L2（或 L3）每个取值

**映射**：参考报告的「关系推理」「汉字理解」等 ≈ 我们的 L2 或 L3 取值

#### 1.3 典型案例增强：维度通过率表 + 结构化失分点

**目标**：每个案例增加「本题目各模型 D2/D3/D4/D5 通过率」及基于 rubrics_check 的失分点列表。

**实现**：

- 在 `analyze_single_case` 中已获取 `dimension_info`（来自 row_level_df）
- 在 case 输出中增加 `dimension_pivot`（题目×模型×维度的通过率）
- 失分点分析：从 `parse_rubrics_check_detailed` 提取 FAIL 的检查点，按 D 维度分组输出，与现有 AI 分析互补

---

### 阶段二：中等成本（依赖约束库/评估结构）

#### 2.1 约束级别分析

**前提**：题目与约束的关联可解析（evaluation_criteria 或约束库）。

**目标**：

- 约束效能分级：A（50%+ 模型失分）、B（30–50%）、C（30% 以下）、D（全满分）
- 约束类型汇总：数量篇幅、边界范围、流程步骤、格式形式 的失分率
- 最具挑战性约束 Top10

**实现**：

- 若 `evaluation_criteria` 可解析为「约束 ID → 类型」：按约束聚合 rubrics_check 的 PASS/FAIL，计算失分率
- 若约束库（constraint_library）有题目→约束映射：同上
- 若无细粒度约束 ID：可退化为「按 D 维度 + L2 约束类型」做汇总（如 D2 流程步骤、D4 格式形式）

#### 2.2 约束失分详情表（案例级）

**目标**：每个案例输出「约束 | 类型 | 满分 | 各模型得分 | ✓/❌」表。

**实现**：

- 需要 evaluation_criteria 或 rubrics 与约束条目的对应关系
- 若 rubrics_check 的 key（如 D2_1）可映射到「约束内容」「类型」「满分」：则可构建该表
- 否则用「D2_1: PASS/FAIL」等形式替代，表头为「检查点 | 维度 | 模型A | 模型B | ...」

---

### 阶段三：总结与建议（可配置或半自动）

#### 3.1 模型能力梯队

**目标**：如「第一梯队 90+ 分」「第二梯队 80–90」「第三梯队 &lt;80」。

**实现**：根据 overall_ranking 的「平均分」分段，可配置阈值（如 90、80、70），在报告中输出梯队划分及简要说明。

#### 3.2 意图级别深度剖析

**目标**：归纳各意图维度下冠军模型及原因（如「汉字理解 GLM 占优」）。

**实现**：基于意图级分析表，取每意图的最佳/最差模型，用模板或简短 LLM 调用生成洞察句。

#### 3.3 约束挑战深度解析

**目标**：解释「全模型失分约束」的含义与改进方向。

**实现**：从约束级别分析中筛选 A 级约束，结合约束内容与类型，输出「可能原因」与「改进建议」的模板文本。

#### 3.4 场景化模型选择指南

**目标**：如「法律文书分析 → Claude」「古诗词理解 → GLM」的推荐表。

**实现**：基于 L2/L3 意图与模型表现，配置「意图 → 推荐模型」映射表，或根据各意图最佳模型自动生成。

#### 3.5 模型能力提升路径

**目标**：针对各模型的改进建议（如「GPT-4o 需加强约束遵循」）。

**实现**：结合梯队、维度得失分、典型 Badcase，用模板或 LLM 生成针对每个模型的 2–3 条建议。
