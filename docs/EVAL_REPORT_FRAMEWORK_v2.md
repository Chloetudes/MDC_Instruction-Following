# 大模型指令遵循评测报告结构框架 v2

> 基于行业评测报告规范与业务洞察导向，草拟的评测报告撰写架构。重点放在业务分析与决策支持，避免堆砌细碎统计。

---

## 一、报告撰写原则（最佳实践对照）


| 原则            | 说明                             | 本框架落地              |
| ------------- | ------------------------------ | ------------------ |
| **执行摘要可独立阅读** | 3–5 页内覆盖目的、方法、核心结论、建议，读者无需翻阅全文 | 第 2 章设为独立摘要        |
| **结论必须有数据支撑** | 每条结论对应具体发现，可溯源到表格/图表           | 每章「洞察」段引用具体指标      |
| **建议可执行**     | 建议面向决策者，明确行动项与责任人              | 第 7 章分角色给建议        |
| **结构层次清晰**    | 总-分-总：先结论后细节，附录放全量表格           | 主报告精简，附录放 Excel 全表 |
| **避免信息过载**    | 主报告突出关键指标，细表进附录                | 主报告每章仅 1–2 张核心表    |

## 二、核心指标优先级（必填 vs 可选）

### 2.1 必填指标（支撑业务结论）


| 指标                  | 用途                                    | 数据来源                               |
| ------------------- | ------------------------------------- | ---------------------------------- |
| **综合排名**            | 模型整体能力排序                              | overall_ranking                    |
| **主分数**             | 维度加权均分（D5=50%, D2=22%, D3=20%, D4=8%） | overall_ranking                    |
| **公开 vs 自建均分差**     | 验证自建题目难度与区分度                          | source_gap_summary                 |
| **专家纠偏排名**          | 专家采样校正后的可靠排名                          | corrected_ranking                  |
| **模型能力分档**          | 三档/四档便于选型                             | model_tiers                        |
| **D2 vs D4 通过率对比**  | 佐证「重格式、轻逻辑」                           | dimension_ila_df / dimension_pivot |
| **L1 高区分度题目数**      | 各意图类型的典型高区分题量                         | l1_high_distinction_stats          |
| **ILA 完全通过率**       | 全检查点 PASS 占比，反映严谨程度                   | full_pass_summary                  |
| **人机一致性（斯皮尔曼/ICC）** | 自动打分可靠性                               | model_expert_overall               |


### 2.2 可选指标（按需展示）


| 指标              | 用途          |
| --------------- | ----------- |
| 分数分布（0–20 分段占比） | 看模型是否偏高分/低分 |
| 厂商版本递进          | 同一厂商内版本迭代效果 |
| 思考 vs 非思考模型均分差  | 思考链对指令遵循的影响 |
| 难度等级×来源分布       | 题目难度结构      |
| 各 L1 专家纠偏排名     | 意图细分选型      |
| 价值题目 TOP20      | 典型案例选题依据    |
| **厂商排名**、**厂商系列模型均分** | 厂商维度柱状图、选型对比 |
| **L1×source 均分**     | 自建 vs 公开 在 L1 层面分数差异 |
| **L1 分差最大/最小题目** | 各任务类型模型表现差异总结 |

## 三、报告结构框架（含数据映射）

### 封面与元信息

- **内容**：报告标题、评测周期、模型数、题目数、数据来源摘要
- **数据**：`model_count`, `task_count`, `eval_count`, `public_cnt`, `self_cnt`（含 H/R/HM/M 细分）

---

### 第 1 章 执行摘要（3–5 页，可独立成文）

**1.1 评测目的与背景**

- 静态文案（可配 `report_content.EXEC_SUMMARY_BACKGROUND`）
- 一句话：本次评测旨在……，通过……框架，实现……目标

**1.2 评测对象与数据**

- 模型数、题目数、公开/自建题数
- 数据来源：`source_stats`（按 source 分组题目数、均分、标准差）
- 关键结论：**公开 vs 纯人工自建均分差**（核心指标）→ `source_gap_summary`

**1.3 核心结论（3–5 条，每条 1–2 句）**

- 例：自建题目上模型均分较公开低 X 分，区分度更佳
- 例：D2（流程步骤）通过率普遍低于 D4（格式），印证重格式轻逻辑
- 例：专家纠偏后 TOP3 为 A、B、C；整体人机排名一致性斯皮尔曼 ρ=0.96
- 数据：`overall_ranking` TOP5、`corrected_ranking` TOP5、`source_gap_summary`、`model_expert_overall` 斯皮尔曼

**1.4 主要建议**

- 模型选型：优先考虑专家纠偏排名与 L1 能力匹配
- 训练优化：强化 D2/D3/D5 等薄弱维度
- 数据构建：增加高区分度自建题，减少 M 合成题偏高分现象

---

### 第 2 章 评测方法论（精简，1–2 页）

**2.1 评测框架**

- 三级分类：L1 指令结构、L2 约束功能、L3 约束类型
- 评分体系：AdvancedIF Rubrics 五维度（D1 业务理解、D2 流程步骤、D3 边界范围、D4 格式形式、D5 内容质量）
- 主分数权重：D5=50%, D2=22%, D3=20%, D4=8%
- 辅助指标：CLA 约束通过率、ILA 完全通过率

**2.2 数据来源说明**

- 公开：to_b、nlp_* 等
- 自建：H（人工）、R（真实来源）、HM（人机协作）、M（模型合成）
- 假设：纯人工自建更能反映业务难度；公开题区分度相对较低

**2.3 专家纠偏机制**

- 有专家打分的题目上，用专家分替换模型自动分后重算均分
- 专家覆盖率、纠偏前后排名变化

---

### 第 3 章 模型能力概览（业务洞察为主）

**3.1 综合排名与分档**

- **表格**：`overall_ranking`（排名、模型、平均分、标准差、最高/最低分）— 主报告仅展示 TOP20，全表放附录
- **分档**：`model_tiers`（第一/第二/第三/第四梯队），便于选型沟通

**3.2 关键洞察（文字段，配 1 张图）**

- L1×模型热力图：`replies_with_question` 按 L1 聚合各模型均分
- 洞察示例：某 L1（如逻辑推理）上模型方差最大，区分度最好；某 L1 上各模型普遍偏低，需重点关注

**3.3 主分数与完全通过率**

- **表格**：`full_pass_summary.per_model_df`（完全通过数、评测数、完全通过率%）
- 洞察：整体 ILA=X%，TOP 模型 ILA=Y%，说明全检查点 PASS 的题目占比低，指令遵循仍有提升空间

**3.4 维度能力摘要**

- **表格**：`dimension_ila_df` 或 `dimension_pivot` 摘要（仅 D2/D3/D4/D5 均值或各模型一行）
- 洞察：D2、D3、D5 通过率明显低于 D4，印证「重格式、轻逻辑」

---

### 第 4 章 分维度与分意图能力洞察

**4.1 L1 意图能力概览**

- **表格**：`l1_high_distinction_stats`（L1、总题目数、高区分度题目数、占比%）
- **文字**：`l1_capability_summaries`（各 L1 的 2–3 句能力总结，来自 LLM 分析）
- 洞察：哪些 L1 有足够高区分度题目支撑分析；各 L1 上最佳/最差模型及差距

**4.2 意图级别分析（精选）**

- **表格**：`intent_level_analysis`（意图、题目数、平均分数方差、最佳/最差模型）— 仅展示方差 TOP 5–10
- 洞察：方差大的 L1 区分度高；方差小的 L1 各模型表现趋同

**4.3 难度与数据来源**

- **表格**：`difficulty_dist_df`、`model_difficulty_df`（按难度等级各模型均分及分差）
- 洞察：易题与难题分差大的模型，在难题上下滑明显

---

### 第 5 章 专家视角与可靠性

**5.1 专家纠偏排名**

- **表格**：`corrected_ranking`（排名、模型、纠偏后均分、原始均分、差异、专家覆盖率）
- 洞察：纠偏前后排名变化；专家采样下的可靠排序

**5.2 专家评测题目榜单（有专家打分的子集）**

- **表格**：`rankings.expert_only`（专家题上按专家均分排名）
- 洞察：与自动打分排名的差异，供厂商交叉验证

**5.3 人机一致性**

- **指标**：`model_expert_overall`（皮尔逊、斯皮尔曼、ICC、MAE）
- **表格**：`model_expert_detail` 或 `model_ranking_summary`（各模型与专家一致性排名）— 主报告可只展示 TOP/BOTTOM 各 5 个
- 洞察：整体一致性较好（斯皮尔曼>0.8），但部分模型 MAE 偏高，自动分需谨慎解读

---

### 第 6 章 典型案例与价值题目

**6.1 价值题目 TOP20**

- **表格**：`top20_questions`（qid、L1、L2、数据来源、模型均分、区分度_D 值、最佳/最差模型）
- 洞察：高区分度题目多集中在哪些 L1、哪些来源

**6.2 典型案例（3–5 个代表）**

- 按 L1 选取 1 个代表案例，含：题目摘要、各模型得分、维度表现、专家意见、AI 综合评估
- 数据：`case_analyses`，按 `score_range` 或 L1 覆盖度筛选

---

### 第 7 章 结论与建议

**7.1 假设验证**

- 自建题目是否更能区分模型？→ 公开 vs 自建均分差、方差比
- 模型是否重格式轻逻辑？→ D2 vs D4 通过率对比
- 专家纠偏是否改变排名？→ 纠偏前后对比

**7.2 分角色建议**

- **选型方**：结合 L1 能力总结与专家纠偏排名，按场景选模型
- **训练方**：加强 D2/D3/D5 相关约束的训练与评测
- **数据方**：增加高区分度自建题（H/R/HM），控制 M 合成题比例

**7.3 场景化选型指南（可选）**

- `scenario_guide`：按 L1/难度/来源给出推荐模型

---

### 附录：全量统计表（供查阅）

- 多维度排名（overall、L1、L2、L3、source、difficulty）
- 专家纠偏排名、L1 专家纠偏排名
- 难度等级分析
- 完全通过率 ILA、D1–D5 各维度 ILA
- 按 Source 分组得分
- 意图级别分析（完整）
- 维度得失分统计
- 分数分布
- 约束类型分析、约束效能
- **厂商排名**、**厂商系列模型均分排行**（见第五节）
- **L1×数据来源均分**、**L1 分差最大/最小题目**（见第五节）
- 厂商版本递进、思考 vs 非思考对比
- 价值题目 TOP20、题目完整分析、L1 高区分度题目统计
- 人工校验分析（如有）
- 指标定义说明

## 五、Excel 统计表计算逻辑与占位符

> 以下为 `analysis_report.xlsx` 中新增统计项的具体计算逻辑、数据来源与输出占位符，供报告生成与框架对接。

### 5.1 厂商相关统计

| 统计项 | 计算逻辑 | 数据来源 | 输出占位符 | 输出位置 |
| ------ | -------- | -------- | ---------- | -------- |
| **厂商排名** | 按 `_get_model_vendor(model)` 识别厂商；每厂商取**均分最高**的模型代表该厂商；按均分降序排序 | `overall_ranking` | `vendor_ranking_df` | Sheet `1_厂商与L1分析` —【厂商排名】 |
| **厂商系列模型均分排行** | 在 `overall_ranking` 上为每模型增加 `厂商` 列（同上识别逻辑）；保持原排名顺序；便于绘制柱状图（横轴=模型，纵轴=均分） | `overall_ranking` | `vendor_series_model_df` | Sheet `1_厂商与L1分析` —【厂商系列模型均分排行】 |

**厂商识别占位符**（`_VENDOR_PATTERNS`）：Qwen / GLM / Claude / GPT / Gemini / DeepSeek / Kimi / Doubao / MiniMax / Ernie / Hunyuan / Grok / Yi / LongCat；未匹配则归为「其他」。

---

### 5.2 L1 × 数据来源 均分

| 统计项 | 计算逻辑 | 数据来源 | 输出占位符 | 输出位置 |
| ------ | -------- | -------- | ---------- | -------- |
| **L1×source 均分** | 按 `(L1, source_group)` 分组，对 `eval_score`（或 `ranking_score`）取**均值**；**不区分模型**；转置为 L1=行、source_group=列 | `replies_with_question` | `l1_source_mean_df` | Sheet `1_厂商与L1分析` —【L1×数据来源 均分】 |

用途：纵轴=L1，横轴=source（公开/自建），展示自建与公开在各 L1 层面的分数差异。

---

### 5.3 L1 分差题目统计

| 统计项 | 计算逻辑 | 数据来源 | 输出占位符 | 输出位置 |
| ------ | -------- | -------- | ---------- | -------- |
| **L1 分差概览** | 分差 = 该题**最高分 − 最低分**（跨模型）；按 L1 汇总：题目数、分差最大/最小/平均、代表 qid | `item_analysis_df`（含 `分数范围`、`L1`、`qid`） | `l1_gap_results.summary` | Sheet `1_厂商与L1分析` —【L1各维度分差概览】 |
| **L1 分差最大题目 TOP10** | 按 L1 分组，按 `分数范围` 降序，每 L1 取前 10 题 | `item_analysis_df` | `l1_gap_results.gap_max[L1]` | Sheet `1_厂商与L1分析` —【{L1} 分差最大题目 TOP10】 |
| **L1 分差最小题目 TOP10** | 按 L1 分组，按 `分数范围` 升序，每 L1 取前 10 题 | `item_analysis_df` | `l1_gap_results.gap_min[L1]` | Sheet `1_厂商与L1分析` —【{L1} 分差最小题目 TOP10】 |

**占位符说明**：
- 分差大：模型在该题上表现差异大，区分度高，适合作为标杆题
- 分差小：模型表现趋同，区分度低，可关注是否题目过易或约束不足

---

### 5.4 统计逻辑汇总（速查）

```
厂商排名     = overall_ranking → 按厂商分组 → 取每厂商最强模型 → 按均分排序
厂商系列排行 = overall_ranking → 增加厂商列 → 按均分排序（便于柱状图）
L1×source   = replies_with_question → groupby(L1, source_group) → mean(score)，不区分模型
L1分差概览   = item_analysis_df.分数范围 → 按L1汇总 max/min/mean 及代表qid
L1分差最大题 = item_analysis_df → 按L1分组 → 按分数范围降序 → head(10)
L1分差最小题 = item_analysis_df → 按L1分组 → 按分数范围升序 → head(10)
```
