# 专家出题统计 — 输入输出效果预览

## 一、输入

### 1. 题目表 `questions_complete_prof.xlsx`（或首次用 `questions_prof.xlsx`）

**路径**：`outputs/<project_id>/questions/questions_complete_prof.xlsx`

| 列名 | 说明 | 示例 |
|------|------|------|
| qid | 题目ID | 1, 2, 3, … |
| **专家** | 出题人/专家名（统计按此列按 qid 对应到回复） | 张三, 李四, 王五, … |
| L3 | 分类/标签 | 指令遵循 |
| query | 题目指令内容 | 请根据以下要求… |
| human_rubrics | 人工评分要点 | 要点1… |
| standard_rubrics | 标准评分要点 | 标准1… |
| reference | 参考答案（可后续由 generate_references 生成） | 参考答案内容… |
| source | 题目来源 | 自建 |
| reply1_model | 回复1 模型名 | model_a |
| reply1 | 回复1 内容 | 模型回复文本… |
| reply1_score | 回复1 专家打分 | 75 |
| reply1_reason | 回复1 专家理由 | 理由… |
| reply2_model | 回复2 模型名 | model_b |
| reply2 | 回复2 内容 | … |
| reply2_score | 回复2 专家打分 | 80 |
| reply2_reason | 回复2 专家理由 | … |
| evaluation_criteria | 评估标准（可由 generate_criteria 生成） | 1. 维度一… 2. 维度二… |

说明：**专家** 列必填且与 qid 一一对应，统计时通过 qid 与回复表关联；其他列按需保留，写入 complete 表时都会保留。

---

### 2. 回复表 `replies_prof.xlsx`

**路径**：`outputs/<project_id>/replies/replies_prof.xlsx`

| 列名 | 说明 | 示例 |
|------|------|------|
| qid | 题目ID（与题目表一致） | 1, 2, 3, … |
| model | 模型名 | ref, model_a, model_b, … |
| reply | 该模型在该题下的回复内容 | 回复文本… |
| 专家打分 | 专家对该条回复的分数（有则参与专家统计） | 72, 85, … |
| 出题人 | 可选；为空时由系统按 qid 从题目表「专家」列自动补全 | 张三, 李四, … |
| eval_batch_2 | 裁判模型评估分数（与 batch_id 对应） | 70, 78, … |
| eval_batch_2_raw | 裁判原始输出（含维度通过情况，可选） | JSON 文本… |

说明：**qid + model** 唯一；有 **专家打分** 的行会参与专家统计，**出题人** 可为空，系统会按 qid 从题目表补全。

---

**区块1：专家数据质量** — 每位专家的出题与质量指标（示例列与顺序）

| 诊断 | 简要诊断说明 | — |
| 优化建议 | 改进建议 | — |

**区块2：各批次人机一致性** — 专家打分与各 eval 批次的一致性（如 batch_1、batch_2）。

**区块3：多批次评估可靠性** — 各评估批次的稳定性排名。

**区块4：每位专家各批次人机一致性** — 每位专家在不同评估批次上与裁判的一致性。

## 三、流程小结

```
输入：
  题目表 questions_complete_prof.xlsx（含 qid、专家、query、reply1/reply2、专家打分相关列、evaluation_criteria 等）
  回复表 replies_prof.xlsx（含 qid、model、reply、专家打分、eval_batch_*）

运行：python3 main.py（stages 含 evaluate_replies、analyze_results）

输出：
  reports/analysis_report_prof.xlsx
    → 打开 sheet「专家数据质量与一致性」查看每位专家的出题数、评估数、人机一致性、ref 质量、区分度、数据质量得分与等级等。
```

**要点**：题目表有 **专家** 列且 qid 与回复表一致；回复表有 **专家打分** 和 **eval_*** 列；系统按 qid 自动补全出题人，保证 19 位专家每人都有完整统计行。
